### MvvmBlazor
MvvmBlazor 是一个用于构建 Blazor WebAssembly 和 Blazor Server 应用程序的小型框架。借助其易于使用的 MVVM 模式，您可以提高开发速度，同时最大限度地减少使其工作所需的工作量。
MVVM框架：[https://github.com/klemmchr/MvvmBlazor](https://github.com/klemmchr/MvvmBlazor)

### mudblazor

### photino
Photino 是一个轻量级的开源框架，用于
使用 Web UI 技术构建本机的跨平台桌面应用程序。
Photino.Blazor 建立在Photino 之上。NET 添加 Blazor 功能。
Photino 使用操作系统内置的基于 WebKit 的浏览器控件，适用于 Windows、macOS 和 Linux。Photino 是最轻量级的跨平台框架。与 Electron 相比，Photino 应用程序最多小 110 倍！而且它使用的系统内存也少得多！
官网：[https://www.tryphotino.io/](https://www.tryphotino.io/)
仓库地址：[https://github.com/tryphotino/photino.Blazor](https://github.com/tryphotino/photino.Blazor)

缺点：

- 要求一个人编译本机C++代码才能正常工作
- 不支持arm linux
- 过度设计
- 没有好好维护

### BlazorWebView
一个[WebKitGtkSharp](https://github.com/GtkSharp/GtkSharp) WebView，用于在 Linux 上运行[Blazor Hybrid](https://learn.microsoft.com/en-us/aspnet/core/blazor/hybrid/)应用程序，而无需编译本机库。
Blazor Hybrid 应用程序允许创建一个 Blazor 桌面应用程序，该应用程序使用本地计算机的 .NET 运行时（不是 Blazor WASM），可以完全访问本地资源（不是沙盒），并且不需要通过 Web 服务器（例如 Blazor 服务器）托管. 它就像用 C## 编写的任何其他桌面应用程序一样，但使用 Blazor 和 Web 技术来实现用户界面。
仓库地址：[https://github.com/JinShil/BlazorWebView](https://github.com/JinShil/BlazorWebView)

## 图标库

### ChartJs.Blazor

ChartJs.Blazor 是一个基于 C# 和 Blazor 开发的前端开源图表库，它把 ChartJs 引入到了 Blazor 中, 并提供了开箱即用的实现，并且支持了 Client 和 Server 两种模式。

通过 ChartJs.Blazor，你可以充分利用 Blazor 的强大功能，以一种直观且直接的方式呈现数据，为用户提供出色的数据可视化体验。

仓库地址：https://github.com/mariusmuntean/ChartJs.Blazor
