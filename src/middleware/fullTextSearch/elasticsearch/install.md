## 概述
在使用ES的时候，Elasticsearch 和 Kibana 这两个容器会一起使用。Elasticsearch 用于存储和处理数据，而 Kibana 则负责可视化和分析这些数据，提供更直观和易于理解的数据呈现方式。

## 安装方案

### docker
创建elasticsearch.yml文件
```shell
cluster.name: "docker-cluster"
## http.host: 0.0.0.0

## 网络绑定
network.host: 0.0.0.0

## 以单一节点模式启动
discovery.type: single-node

## 跨域
http.cors.enabled:  true 
http.cors.allow-origin:  "*"
```
需要安装es
```shell
docker run --name es -d  -p 9200:9200 -p 9300:9300 -e "discovery.type=single-node" -e "ES_JAVA_OPTS=-Xms128m -Xmx256m" -v elasticsearch.yml:/usr/share/elasticsearch/config/elasticsearch.yml elasticsearch:7.16.3

#9200是对外端口，9300是es内部通信端口
#-e ES_JAVA_OPTS="-Xms128m -Xmx256m"  Xms初始化时内存栈大小，Xms程序限制占用内存
#-v 把配置文件挂载到es的docker里的配置文件
```
然后就可以通过访问该地址来查看：[http://localhost:9200/](http://localhost:9200/) ，输出结果

```shell
docker run --name kibana -e ELASTICSEARCH_HOSTS=http://localhost:9200 -p 5601:5601 -e "I18N_LOCALE=zh-CN" -d kibana:7.16.3
```

### docker-compose
elasticsearch.yml文件
```yaml
cluster.name: "docker-cluster"
## http.host: 0.0.0.0

## 网络绑定
network.host: 0.0.0.0

## 以单一节点模式启动
discovery.type: single-node

## 跨域
http.cors.enabled:  true 
http.cors.allow-origin:  "*"
```

kibana.yml文件
```yaml
server.name: kibana
## kibana的主机地址 0.0.0.0可表示监听所有IP
server.host: "0.0.0.0"

## 这边设置自己es的地址，
elasticsearch.hosts: [ "http://elasticsearch:9200" ]
## elasticsearch.username: 'kibana'
## elasticsearch.password: '123456'
## 显示登陆页面
monitoring.ui.container.elasticsearch.enabled: true

## 开启中文模式
i18n.locale: "zh-CN"
```

最后docker-compose文件如下
```yaml
version: '3'

services:
  elasticsearch: ## 存储和处理数据   http://用户名:密码@localhost:9200/
    container_name: elasticsearch
    image: elasticsearch:7.16.3
    environment: 
      discovery.type: single-node
      xpack.security.enabled: 'false' ## 验证关闭
      ES_JAVA_OPTS: -Xms128m -Xmx256m ## Xms程序限制占用内存，Xms初始化时内存栈大小
      TZ: Asia/Shanghai
    ports:
      - 9200:9200 ## 对外端口
      ## - 9300:9300 ## es内部通信端口
    restart: always
    volumes:
      - ./config/es/elasticsearch.yml:/usr/share/elasticsearch/config/elasticsearch.yml
      ## - /data/es:/usr/share/elasticsearch/data
    networks: 
      - my-bridge

  kibana: ## 可视化和分析数据
    container_name: kibana
    image: kibana:7.16.3
    ## restart: always
    ports:
      - 9400:5601
    depends_on:
      - elasticsearch
    environment:
      - ELASTICSEARCH_HOSTS=http://elasticsearch:9200
      - TZ=Asia/Shanghai
    privileged: true
    volumes:
      - ./config/kibana/kibana.yml:/usr/share/kibana/config/kibana.yml
      - /home/volumes/kibana/log:/usr/share/kibana/log
    networks: 
      - my-bridge


networks: 
  my-bridge:
    external: true ## 表示一个已经存在的外部网络，不需要使用docker再次创建，可以将多个docker-compose文件连接到同一个网络中
    driver: bridge      
```
然后就可以通过访问该地址来查看：[http://localhost:9200/](http://localhost:9200/) es服务，使用[http://localhost:9400/](http://localhost:9400/)来查看kibana服务

## 安装分词器

### IK分词器
下载地址：[https://github.com/medcl/elasticsearch-analysis-ik/releases](https://github.com/medcl/elasticsearch-analysis-ik/releases)

 把文件下载后，解压复制到es部署文件的plugins文件夹下，并把文件夹的名称改为ik，必须要叫ik，然后以挂载文件夹的方式把文件映射进去

## 问题

### 重置ES密码
```shell
## 重置 elastic 用户密码
docker exec -it elasticsearch-8.8.1 /usr/share/elasticsearch/bin/elasticsearch-reset-password -uelastic

## 提示如下 输入 y：
This tool will reset the password of the [elastic] user to an autogenerated value.
The password will be printed in the console.
Please confirm that you would like to continue [y/N]

## 输入后显示结果 xxxxxx 即为密码
Password for the [elastic] user successfully reset.
New value: xxxxxx
```

### 重置Kibana密码
```shell
## 重置 kibana_system 用户密码
docker exec -it elasticsearch-8.8.1 /usr/share/elasticsearch/bin/elasticsearch-reset-password -ukibana_system

## 提示如下 输入 y：
This tool will reset the password of the [kibana_system] user to an autogenerated value.
The password will be printed in the console.
Please confirm that you would like to continue [y/N]

## 输入后显示结果 xxxxxx 即为密码
Password for the [kibana_system] user successfully reset.
New value: xxxxxx

```
