## 概述

大模型AI（Large Model AI）是指具有大量参数和复杂结构的人工智能模型。这些模型通常基于深度学习技术，能够处理大量数据并从中学习复杂的模式和关系。大模型AI通过其庞大的参数量和深层次的网络结构，能够在多个领域中表现出卓越的性能。



LLM(大语言模型) 是在大量数据上训练的深度学习模型，能够理解和生成类似人类的文本。 这些模型可以执行各种任务，例如文本完成、摘要、分类和参与对话。

## 常见概念

### AI Agent

诞生于上世纪80年代，相认一样交互的智能软件。

在人工智能和计算机科学领域，"Agent"（智能体）指的是能够感知环境并根据感知到的信息做出决策和行动的系统或软件实体。Agent 的概念广泛应用于各种领域，包括机器人技术、自然语言处理、游戏开发等。

Agent由三个组件组成： **大型语言模型 （LLM）、** 可以使用的一组**工具**以及提供说明的**提示** 。

#### 特点

自主性：AI Agents 能够独立执行任务，无需持续的人工干预或输入。

感知：Agents 功能通过各种传感器（例如摄像头或麦克风）感知并解释他们所处的环境。

高响应性：AI Agents 可以评估环境并做出相应响应以实现其目标。

推理和决策：AI Agents 是智能工具，可以分析数据并做出决策以实现目标。他们使用推理技术和算法来处理信息并采取适当的行动。

学习：他们可以通过机器、深度和强化学习元素和技术来学习和提高自己的表现。

通信：AI Agents 可以使用不同的方法与其他代理或人类进行通信，例如理解和响应自然语言、识别语音以及通过文本交换消息。

面向目标：它们旨在实现特定目标，这些目标可以预先定义或通过与环境的交互来学习。

### RAG（Retrieval-Augmented Generation）

RAG是一种人工智能框架，旨在通过检索增强生成来提高大语言模型（LLM）的自然语言处理能力。其核心思想是结合检索和生成两种技术：

- **检索（Retrieval）**：从外部知识库中检索与用户查询相关的文档或信息块。这些文档可以是文本、图像或其他形式的数据。
- **生成（Generation）**：利用检索到的信息作为上下文，通过大语言模型生成回答或内容。

RAG通过这种方式，能够有效地解决大模型在知识更新速度慢和知识覆盖范围有限的问题。

### 模型温度Temperature

调整LLM的“温度”（Temperature）参数。这个设置控制着模型输出的随机性。低温度（接近0）带来的是精确、可预测的结果，适合需要稳定性的任务；高温度（接近1）则引入更多变化和创意。在0到1之间调整，就是在“严谨”与“发散”之间寻找平衡。根据Cohere的经验，对于AI Agent的调用场景，0到0.3的温度通常效果最佳——越需要精确和可预测，温度就该设得越低。

## Rag应用创建步骤

如果你想要自己创建一个RAG应用，可以按照以下流程进行：

### 1. **数据准备阶段**

- **准备文档**：收集作为RAG系统基础输入数据的文档。这些文档可以来自企业内部的知识库、公开的网页或其他数据源。
- **文档处理**：将文档转换为纯文本数据。如果文档是图像格式（如PDF中的文字），可以使用OCR技术提取文本。
- **文本分割**：将文本分割成多个小块（chunks），以便更高效地处理和检索信息。
- **文本向量化**：使用嵌入模型（embedding model）将每个文本块转换为向量形式，捕捉其语义信息。
- **创建向量数据库**：选择一个向量数据库（如FAISS、Chromadb等），将生成的向量存储其中，以便后续检索。

### 2. **应用阶段**

- **用户查询处理**：当用户输入问题时，系统将其转换为与文档块相同的向量维度，确保问题和文本块位于同一向量空间。
- **检索最相关信息**：在向量数据库中搜索与用户问题向量语义上最相似的文本块。
- **生成回答**：将用户问题和检索到的信息结合，构建提示模板，输入到大语言模型（LLM）中，生成最终答案。

### 3. **技术实现步骤**

- **安装必要的库**：例如LangChain、Chromadb、OpenAI等。
- **加载和配置环境**：配置环境变量，如API密钥。
- **加载文档**：使用文档加载器（如LangChain的WebBaseLoader）从目标地址读取数据。
- **文档拆分和向量化**：使用文本拆分器和嵌入模型将文档拆分为小块并转换为向量。
- **创建向量数据库和检索器**：使用向量数据库创建索引，并配置检索器。
- **处理用户查询和生成回答**：将用户查询向量化，检索最相关信息，并结合LLM生成答案
